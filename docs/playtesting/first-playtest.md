# 첫 플레이테스트 체크리스트

기본 스테이지의 재미와 모바일 사용성을 분리해서 기록한다. 한 번의 인상으로 결론 내리지 말고 같은 빌드에서 seed를 바꿔 최소 3회 플레이한다.

자동 테스트와 빌드만으로 이 문서의 실기기 항목을 완료 처리하지 않는다. 현재 검증 상태와 설치 절차는 [모바일 빌드와 기기 검증](../ops/mobile-build-validation.md)을 따른다.

## 빌드와 기기 식별

- [ ] Git 커밋 SHA와 APK 또는 archive 식별자를 기록했다.
- [ ] 앱 버전, 기기 모델, OS 버전을 기록했다.
- [ ] 화면 크기, 노치/카메라 홀 유무, 가로 방향을 기록했다.
- [ ] 기존 앱 위에 업데이트했는지 새로 설치했는지 기록했다.
- [ ] 실행 직전과 문제 재현 직후의 기기 로그를 보관할 방법을 확인했다.

## 세션 정보

- [ ] 날짜와 빌드/커밋을 기록했다.
- [ ] 기기, 운영체제, 화면 방향과 해상도를 기록했다.
- [ ] 사용한 seed와 최종 결과(승리/패배)를 기록했다.
- [ ] 시작부터 결과 화면까지 걸린 시간을 기록했다.

## 진행과 난이도

- [ ] 시작 재화로 소환 3회를 막힘없이 수행했다.
- [ ] 웨이브 1, 2, 3의 난이도를 각각 1~5점으로 평가했다.
- [ ] 지루해진 시점 또는 갑자기 어려워진 시점을 웨이브와 시간으로 기록했다.
- [ ] 승리 또는 패배 원인이 플레이 중 이해됐는지 기록했다.
- [ ] 한 판 길이가 다시 플레이하고 싶은 범위인지 기록했다.

## 무작위성 체감

- [ ] 소환된 타워 순서를 기록했다.
- [ ] 서로 다른 seed에서 결과가 충분히 달라 보였는지 기록했다.
- [ ] 같은 타워가 연속 등장했을 때 불공정하거나 답답하게 느껴졌는지 기록했다.
- [ ] 어떤 타워가 등장해도 진행 가능했는지 기록했다.

## 조작과 화면

- [ ] 소환 버튼이 한 번의 탭에 한 번만 반응했다.
- [ ] 화면 모서리와 전투 영역을 탭해도 의도하지 않은 소환이나 UI 입력이 발생하지 않았다.
- [ ] 체력, 재화, 웨이브, 소환 가능 여부를 즉시 이해할 수 있었다.
- [ ] 노치와 화면 가장자리 때문에 HUD 또는 버튼이 잘리지 않았다.
- [ ] 홈 화면 이동과 복귀를 3회 반복해도 세션이 중복되거나 멈추지 않았다.
- [ ] 화면 잠금과 해제 뒤에도 전투와 UI가 한 번씩만 갱신됐다.
- [ ] 승리 화면에서 전투 입력이 차단되고 결과가 명확히 표시됐다.
- [ ] 패배 화면에서 전투 입력이 차단되고 결과가 명확히 표시됐다.
- [ ] 결과 화면의 재시작이 이전 적, 타워, 투사체 없이 새 판을 시작했다.

## 플랫폼 완료 판정

- [ ] Android ARM64 실제 기기에서 설치와 첫 실행이 성공했다.
- [ ] Android에서 위 조작·화면·승패·재시작 항목을 모두 확인했다.
- [ ] iOS Xcode archive와 서명이 성공했다.
- [ ] iPhone 또는 iPad에서 위 조작·화면·승패·재시작 항목을 모두 확인했다.
- [ ] 실행하지 못한 플랫폼 항목은 필요한 기기나 계정과 함께 미검증으로 기록했다.

## 버그 기록

각 버그는 별도 항목으로 다음을 적는다.

- 발생 시점과 seed
- 재현 순서
- 기대 결과와 실제 결과
- 재현 빈도
- 스크린샷 또는 로그

## 개선 의견

버그와 섞지 말고 다음 질문에 답한다.

- 가장 재미있었던 순간은 무엇인가?
- 가장 답답하거나 의미 없었던 선택은 무엇인가?
- 한 가지만 다음 빌드에서 바꾼다면 무엇인가?
- 타워, 적, 웨이브 중 어떤 콘텐츠를 먼저 늘리고 싶은가?
